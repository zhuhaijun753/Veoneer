This document aims to describe the vehicle interface for the Active Safety Platform. The physical layer used for the interface is CAN-FD.

2 Front Looking ADAS Camera (FLAC)

End-2-End protection is performed using Autosar Profile 2, Autosar E2E.

2.2 Classified Detections

This message contains Classified Detections based on grouped classificiations without tracking.

Maximum number of detections: 50

Maximum bandwidth usage: 0.140 Mbit/s

2.2.1 Header

  • Crc (cdhCrc): According to Autosar Profile 2
  • Counter (cdhCounter): According to Autosar Profile 2
  • ListIndex (cdhListIndex): Used to synchronize with content messages
  • FrameSyncNumber (cdhFrameSyncNumber): Used to synchronize with VisionFrameInformation
  • ListLength (cdhListLength): The number of entries in the resulting array
  • Latency (cdhLatency): Time from sensor data capture to output on vehicle interface

Data ID to be defined.

2.2.2 Content

  • Crc (cdCrc): According to Autosar Profile 2
  • Counter (cdCounter): According to Autosar Profile 2
  • ListIndex (cdListIndex): Used to synchronize with header and other content messages
  • Index (cdIndex): Describes where to place the data in the resulting array
  • Object[A/B/C/D] (replace A on the end below)
    • LeftAclc
      • X (cdLeftAclcXA): X position for lower-left boundary of detection in ACLC
      • Y (cdLeftAclcYA): Y position for lower-left boundary of detection in ACLC
      • Z (cdLeftAclcZA): Z position for lower-left boundary of detection in ACLC
    • RightAclc
      • X (cdRightAclcXA): X position for lower-right boundary of detection in ACLC
      • Y (cdRightAclcYA): Y position for lower-right boundary of detection in ACLC
      • Z (cdRightAclcZA): Z position for lower-right boundary of detection in ACLC
    • Height (cdHeightA): Height of detection
    • Continuation (cdContinuationA): Describes the direction the detection continues in, if any.
    • Class (cdClassA): Describes the type of the detection
    • Probability (cdProbabilityA): Existance probability of detection
    • UsedDisparity (cdUsedDisparityA): Describes if Disparity was used to generate the distance to the detection

Data ID to be defined.

2.3 FlacStatus

This message describes the current status of the Front Looking ADAS Camera (FLAC).

  • Status (flacStatus): Describes the state of the unit

2.4 InterfaceVersion

This message is used to inform the users which version of the interface protocol is used. The format of this message shall never change since this breaks backwards compatibility. Using the information in this message one can find the correct interface specification.

Maximum bandwidth usage: <0.001 Mbit/s

  • MajorVersion (interfaceMajorVersion): Change in number indicates a breaking change
  • MinorVersion (interfaceMinorVersion): Change in number indicates a version compatible change

2.13 StereoDetections

Maximum number of detections: 80

Maximum bandwidth usage: 0.32 Mbit/s

2.13.1 Header

  • Crc (sdhCrc): According to Autosar Profile 2
  • Counter (sdhCounter): According to Autosar Profile 2
  • ListIndex (sdhListIndex): Used to synchronize with content messages
  • FrameSyncNumber (sdhFrameSyncNumber): Used to synchronize with VisionFrameInformation
  • ListLength (sdhListLength): The number of entries in the resulting array
  • Latency (sdhLatency): Time from sensor data capture to output on vehicle interface

Data ID to be defined.

2.13.2 Content

  • Crc (sdCrc): According to Autosar Profile 2
  • Counter (sdCounter): According to Autosar Profile 2
  • ListIndex (sdListIndex): Used to synchronize with header and other content messages
  • Index (sdIndex): Describes where to place the data in the resulting array
  • Object[A/B/C] (replace A on the end below)
    • LeftAclc
      • X (sdLeftAclcXA): X position for lower-left boundary of detection in ACLC
      • Y (sdLeftAclcYA): Y position for lower-left boundary of detection in ACLC
      • Z (sdLeftAclcZA): Z position for lower-left boundary of detection in ACLC
    • MiddleAclc: Set to arithmetic mean value of left and right point if detection is not an L-Shape
      • X (sdMiddleAclcXA): X position for lower-middle boundary of detection in ACLC
      • Y (sdMiddleAclcYA): Y position for lower-middle boundary of detection in ACLC
      • Z (sdMiddleAclcZA): Z position for lower-middle boundary of detection in ACLC
    • RightAclc
      • X (sdRightAclcXA): X position for lower-right boundary of detection in ACLC
      • Y (sdRightAclcYA): Y position for lower-right boundary of detection in ACLC
      • Z (sdRightAclcZA): Z position for lower-right boundary of detection in ACLC
    • DirectionValid (sdDirectionValidA): If set end points are valid. If not set end points shall be seen as the bounding box created from the points, i.e. when not set it describes the uncertainty.
    • Height (sdHeightA): Height of detection
    • IsLShape (sdIsLShapeA): Describes if all three points should be used to form an L-Shape or if only left and right point shall be used.

Data ID to be defined.
