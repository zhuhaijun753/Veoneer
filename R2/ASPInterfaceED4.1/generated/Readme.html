1 Purpose

This repository is used to synchronize and exchange interfaces between the different components that make up System Products.

2 Interface forum

The interface forum is a teams channel used to communicate around interfaces. The teams channel can be found here.

On the wiki page the Participants are described to make it easy to understand who to interact with for what.

3 Repository structure

Each organization provide their interfaces within their subfolder under Interfaces. Each organization can create their own directory structure in their folder. It is however important that all interfaces are compatible so enums, structures, etc should have unique names

  • ActuatorSupport - Contains support functionality for interfacing with different actuators. E.g. converting steering request to steering torque request.
  • Integration - Documentation needed to integrate in the System Core environment and APIs for interacting with services (debug logging, etc) on Aurix
  • RadarCore - Contains RadarCore interfaces
  • SystemCore - Contains System Products bus interfaces as well as wrapper for debug data and internal data
  • Tools - Contains Protobuf implementation and C code generator for target
  • VisionCore - Contains Vision Core interfaces
  • Zenuity - Contains Zenuity interfaces

4 Documentation

  • Integration/data_interfaces.md - Describes how to package data for internal distribution or debug
  • Integration/transport_protocols_and_physical_interfaces.md - Describes the different debug data distribution options
  • Integration/zenuity_interface_design_description.md - Describes how to integrate in System Products environment

5 Interface generation

generate_all_interfaces.sh can be used to generate all documentation and code for the System Product Protocol interfaces. Results will be placed in the generated folder.

Installation guide can be found in Integration/data_interfaces.md.

6 Markdown to HTML generation

Pandoc can be used to generate html from markdown. Install latest pandoc from Pandoc install page.

pandoc -o zenuity_interface_design_description.html zenuity_interface_design_description.md

6.1 Using inline plantuml images

To be able to generate plantuml images the pandoc-plantuml-filter is used. It can be installed using pip:

pip install pandoc-plantuml-filter
pip install pandoc-fignos

And now the filter can be activated using the this command instead:

pandoc -F pandoc-plantuml -F pandoc-fignos -o zenuity_interface_design_description.html zenuity_interface_design_description.md

To generate html for all md files use the script:

./generate_markdown.sh

Note #1: On windows the PLANTUML_BIN env variable needs to be set to “java -jar <plantuml.jar>”, the jar file can be downloaded from PlantUML download page.

Note #2: If the error “fd:4: hPutBuf: resource vanished (Broken pipe)” occurs there is a problem with the filter and pandoc versions. It might be solved by installing the latest binary from pandoc website instead of using default package management (for example apt)
