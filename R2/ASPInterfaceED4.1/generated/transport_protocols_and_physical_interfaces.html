This document aims to describe the different transport protocols and physical interfaces used for debug data. Data format is described in detail in data_interfaces.

1 Data Transportation

The transport layer is responsible that the information is transferred correctly and safely.

The endpoints of the transport layers shall also be implemented on ASIL level.

Transport protocol properties: | Interface property | Data Type (minimum) | Rationale | | CRC Type | 1 bit (0: QM CRC, 1: ASIL CRC) | Identifies the CRC used to verify the data | | Checksum | U32 | The result of the CRC calculated for the data block | | Sequence Counter | U8 | Verification that all packets are received and in order | | First message | 1 bit | Describes that the data buffer is the start of a new package | | Data Size | U16 | Describes the total length of the data package. Only necessary for the first message. |

2.1.4 RTP

An overview of RTP can be found on Wikipedia.

The RTP layer is used to fragment and reassemble large data packets. One RTP packet contains not more than a single data message. However longer data messages can stretch over multiple RTP packets.

Property Description
Version 2
Padding (P) 0
Extension (X) Depends on if Header Extension is included
CSRC Count (CC) 0
Marker (M) Used to signal first packet. 1 = first part of payload, 0 = additional payload
Payload Type (PT) 101
Sequence Number According to RTP specification
Timestamp Time in micro-seconds
SSRC Transmitter identification (Transmitters)
Header Extension(s) Header Extension Id 155 is used to add additional information

2.1.4.1 Header Extensions

This extension is included for the first RTP packet of a multi packet payload.
